# 气味系统设计文档

> 关二狗是个背尸匠，他的鼻子比眼睛先知道事。本文档把"气味"从散落的过场 flavor 提升为一套**常驻、可学习、值驱动**的感官机制——让阿秀的"香粉味"信号真正立得住，同时给关二狗一个本行工具、给世界一层可玩的感官深度。
> 上级文档：《遭遇系统总设计文档》《身体资源系统设计文档》；世界观依据《世界设定-煞之机理》。
> 关联系统：离死之距 / 三把火（GameDraft `HealthSystem` + HUD）、死亡系绳、知识系统、规矩系统。

---

## 一、设计动机：为什么要这个系统

**问题**：香粉味（阿秀冷信号）目前落地很失败。它只是散落在各处过场里的一句"一缕香粉味飘起"——背尸看一次、义庄看一次，中间**没有任何贯穿的承载 channel** 让玩家把这几次连起来。看过就忘，**立不住**。

**类比（关键）**：三把火（离死之距）为什么立得住？因为它是个**常驻、可读、值驱动的系统**——玩家天天看着它、学会读它，于是"残星"出现时一眼就懂。香粉味要立，就得有同样"玩家天天在用、学会读"的系统。

**一箭三雕**：把气味做成机制，同时解决三件事——
1. **立住香粉味**：它泡在一个玩家熟透的系统里，飘出来就有人读得懂。
2. **给关二狗一个本行工具**：背尸匠的鼻子，是他贴身的感官能力。
3. **扣住阿秀主题**：阿秀靠"读气"通阴（煞机理 §九·补），气味系统正是这条主题的玩家侧粗版。

---

## 二、核心铁律：系统常驻可学习 ≠ 香粉味常见

这是全系统**最重要的一条**，写在最前面防跑偏。

香粉味在冷框架里是**冷·盲目·只在濒死显形·冥冥之中**——刻意**稀有**才特殊（见《阿秀支线-完整剧情与主线实施方案》§8b/§9）。而"做成机制、总能看到"听着像要它**常见**。这俩是反的。

**解法（三把火给的答案）**：
- **气味系统/channel** 做成常驻、可学习的（像三把火那个 HUD 永远在）；
- **香粉味本身仍是这套系统里一个稀有、有意义的"读数"**——就像三把火的"残星"：HUD 天天在，残星只在濒死才出现。

玩家因为**天天用气味系统闻别的东西**，学会了"怎么读气味"；于是香粉味哪天飘出来，他**立刻能读懂**——但它没变成墙纸，还是冷的、特殊的。

> **铁律一句话**：系统让香粉味**可读**，不是让它**常见**。别因为"做成机制"就把它撒得到处都是——冷味一泛滥就废了。

## 三、世界观锚：气味 = 关二狗对"气"世界的粗感知

煞机理里：**气**是弥漫一切的底层场；**阿秀**靠至阴之体**读气**（衣承气 / 至阴感气，§九·补）——这是高灵敏、深层的感知。

关二狗一个粗人背尸匠，读不了深的气，**但鼻子闻得到粗的那一层**：尸臭、阴冷的腥、霉、血。
→ **气味系统 = 关二狗对"气"世界的粗感知探头**（鼻子版、低分辨率版的"读气"）。
→ **香粉味 = 阿秀那口念气，crude 到连他这鼻子都闻得到的那一处**——阿秀读气是手术刀，关二狗闻气是钝口，唯独她那口"不撒手"的念气浓到他都嗅得到。

一套系统同时把"关二狗的本行"和"阿秀的气"焊在一起，主题白赚。**给玩家只演气味，绝不出"场""气""耦合"等机理词**（机理只活在幕后）。

## 四、气味系统是什么

**关二狗随身的一套感官**：在场景里能闻到一组**有来源、有意义**的气味，按强弱/方位呈现。

**气味种类（示例，按玩法用途分，非穷举）**：

| 气味 | 来源 | 提示什么 |
|---|---|---|
| **尸臭** | 腐尸、停尸处 | 附近有死人；**该臭不臭=反常**（不腐尸的恐怖前奏） |
| **阴冷的腥** | 阴气重处、阴宅、井底、乱葬岗 | 这地方"不干净"、方位有阴物 |
| **香火 / 纸钱灰** | 庙、灵堂、丧家、路祭 | 丧俗现场、规矩相关场 |
| **烧酒** | 酒馆、醉汉、酒浸血状态 | 社交/线索；呼应身体资源"酒浸血" |
| **血腥** | 伤口、刚死的、凶案现场 | 暴力刚发生过 |
| **霉 / 土腥** | 久无人住、地窖、棺木 | 废弃、寄柩、年头久 |
| **蛊寨怪味 / 药味** | 下蛊寨子、草药 | 危险饮食警告、药血线索 |
| **★廉价香粉味（+一丝阴冷）** | 阿秀帕子包 / 空香粉盒 | **阿秀冷信号**——稀有冷读数，见 §六 |

**原则**：每种气味**必须有玩法用途**（线索 / 警告 / 规矩 / 恐怖前奏），否则就是带 HUD 的 flavor，重蹈香粉味覆辙。

## 五、玩法用途（气味系统怎么"挣饭吃"）

| 用途 | 怎么玩 |
|---|---|
| **调查线索**（喂知识系统） | 闻出"这具尸该臭不臭=不对"、闻出阴气重的方位、闻出哪间屋"不干净"——气味是低成本的非视觉线索通道 |
| **恐怖前奏 / telegraph** | 进门先闻到尸臭/阴腥 → 玩家知道附近有东西，**比跳脸高级**；该臭不臭的"缺席"更瘆 |
| **规矩 / 遭遇钩子** | 路祭的酒肉味、丧家的香火味牵出规矩场；蛊寨怪味="别乱吃"的天然警告 |
| **状态耦合** | 酒浸血时酒味盖过别的味（判断变钝，呼应身体资源副作用）；阳气低时更易闻到阴腥 |
| **★阿秀信号** | 香粉味在系统里作为一个可被玩家"读"的味出现——见 §六 |

## 六、香粉味在系统里的定位（稀有冷读数·守冷框架）

香粉味**不是**气味系统里随处可闻的环境味，它是这套系统里一个**稀有、冷、有意义的特殊读数**：

- **何时出现**：仍按冷框架——**濒死盲目显形**（背尸坠潭、送货应声/出山口、义庄濒死等），随帕子包走、只认物不认人、非守护。**低血/平时不乱飘。**
- **为什么现在立得住**：玩家已经**天天用气味系统**闻尸臭/阴腥/香火，鼻子被训练过；所以香粉味一出现，他**立刻读得出"这味不对、是那股冷东西"**——而不是像现在这样看过就忘。
- **和旧实现的区别**：旧的是过场里硬塞一句"香粉味飘起"，无 channel、无积累；新的是**系统里一个玩家学过怎么读的冷读数**。表现统一（不再各做各的）。
- **守纪律**：香粉味 + 半句跑调小调仍是阿秀**专属**，环境气味一律用别的味，**绝不借用香粉味/小调**（信号防污染纪律不变）。

## 七、和三把火 / 死亡系绳的耦合（很优雅，别浪费）

两套感官系统在**"该死那一拍"天然咬合**：
- **离死之距残星（濒死）↔ 香粉味起（系绳）**：三把火快灭了，香粉味才盲目飘出来把人薅回（死亡系绳）。
- 玩家分别学会的两个系统，在最关键那一拍交叉印证——三把火告诉他"快没了"，香粉味告诉他"那股冷东西又来了"。
- **阳气低 ↔ 更易闻到阴腥**：身体资源的阳气浓度也可接进气味灵敏度（阳衰则阴味更冲），三系统串成一气。

## 八、表现：HUD 气味指示器（方案 E · 双层·基线+浮现 —— 定稿 2026-06-29）

> **修订记**：早前一版把气味做成"`setSmell` 冒出烟絮、`clearSmell` 烟絮消失"的**触发式**——**错了**。气味是关二狗鼻子的**常驻持续状态**，不是一次性触发。本节定稿改为**方案 E·双层**：常驻基线永远在、气味从基线浮现/沉回。旧实现（`src/ui/HUD.ts` 气味烟 appear/disappear）需按本节重做。

### 8.1 形态：双层（无味＝只剩基线 / 出味＝气缕浮现）

- **正常态（无味）＝只有常驻基线**：HUD 元信息层（与三把火同列）底部一抹**极淡、慢呼吸的中性雾**（横向小团），**没有竖的气缕**。意思是"鼻子在、没闻到特别的"。它**永远在**——这才是"持续状态"，不是空白。
- **出味＝气缕从基线浮现**：一缕**虚边气体**（柔和、边缘糊、越升越散，像热茶上那缕香——不是浓烟、不是实线丝带）从基线升起；基线本身也**微微染上该味的色**。退味＝气缕沉回、基线褪回中性。
- **过渡平滑**：状态切换走淡入淡出 / lerp，**绝不凭空冒、凭空消**。
- 纪律同三把火：不上全屏、不喊注意、纯冥冥之中；**世界层不画气味颗粒**（防污染，见 §十一）。

### 8.2 编码语法：一属性一通道

| 属性 | 视觉通道 |
|---|---|
| **味道·身份**（哪种味） | **颜色**（主）+ **飘法签名**（辅）——见 §8.3 |
| **浓淡·强度** | **密度 + 亮度 + 升起高度 + 体量**（越浓越密越亮越高越粗），**不靠加根数** |
| **沉浮·轻重** | 重味（尸臭/霉）贴地翻涌、升不高；轻味（香火）高升——兼当身份线索 |
| **冷·不对劲（wrong）** | 飘法**违反物理**（逆/卡/拐）——香粉味专属冷读 |
| **逼近**（v1） | 强度**随时间渐涨**＝那股味/危险在靠近（编排侧 `setSmell` 给递增强度即可，无需新机制） |
| **波动·稳/动**（v1） | 不稳的味**明灭跳动**；稳定味匀（一个参数驱动） |
| **方位·来源**（v1） | 气缕整体**拖向/偏向**来源那一侧——玩家"闻出朝哪边"（需 SmellSystem 多带一个方向量 `dir`） |

### 8.3 气味词库（颜色 + 飘法签名）

权威源：`public/assets/data/smell_profiles.json`（编辑器「气味Profile」面板可视化改/实时预览）；本表为设计快照。

| id | 名 | 调性色 | 飘法签名 |
|---|---|---|---|
| `incense` | 香火 | 暖橙 `#d6a24e` | 袅袅直上、几乎不摆、升得高 |
| `yin` | 阴腥 | 青冷 `#6f93a6` | 飘忽发抖、宽摆高频 |
| `corpse` | 尸臭 | 灰绿浊 `#8a9a5a` | 沉坠贴地、慢、升不高（heavy） |
| `blood` | 血腥 | 暗红 `#9a3a3a` | 低冲、中摆 |
| `mold` | 霉·土腥 | 灰褐 `#8f7d59` | 闷在低处、最慢（heavy） |
| `powder` | ★香粉味 | 冷粉白透青 `#cdb8cf` | **飘得不对劲**（wrong）——违反物理，一眼"这味不对" |

> 只有词库内的 id 会渲染；`setSmell` 传未登记 id → 静默不画（扩味种：直接在 `smell_profiles.json` 加一条 profile，编辑器面板即可）。
> **★香粉味守冷框架**：`wrong` 标记、飘法刻意违反物理——这正是它"冷·不对劲"的落点。它是常驻系统里一个**稀有冷读数**（§二/§六），平时不乱飘，只在濒死盲目显形那几拍投放。

### 8.4 嗅 action · 主动闻（v1）

给玩家一个**"嗅"键**：按一下，当前气缕**短暂变浓变清**（强度临时拉高、方位更明确），几秒回落。把气味从纯被动指示升级成**主动玩法**——玩家主动去"读"环境（闻这具尸该臭不臭、闻哪间屋不干净、闻阴气在哪边）。数据驱动：新增 `sniff` 输入/命令，读当前 SmellSystem 状态临时 boost 显示。

### 8.5 驱动链路 + 策划接口

- `SmellSystem` own **持续状态**：`scent`（当前味）+ `intensity`（0–100）+（新增）`dir`（方位）+ `flicker`（稳/动）。状态进 FlagStore、经 EventBus 广播给 HUD 实时重绘。
- 策划命令（与三把火 `setHealth`… 同规范，进编辑器动作下拉）：
  - **`setSmell(scent, intensity, [dir], [flicker])`** —— 设当前持续气味状态。
  - **`clearSmell()`** —— **回落到正常态（基线仍在）**，不是消失。
- **逼近**＝编排侧给递增 `intensity`；**方位**＝给 `dir`；**波动**＝给 `flicker`。
- **调试开关**：游戏 URL 加 `?smellDebug`（DEV-only、平时关），console 里 `__smell(scent,intensity,dir,flicker)` / `__smellSniff()` / `__smellStep(n)` 即时驱动 HUD 气味指示器看效果。

### 8.6 v1 范围（本次定稿 2026-06-29）

核心（色＝味种 / 飘法签名 / 密度·高度·亮度＝浓淡 / 沉浮 / 香粉冷读）**＋ 逼近·渐浓 ＋ 波动·明灭 ＋ 方位·来源 ＋ 嗅 action 主动闻**，四个"其他属性"全做。世界气味颗粒（③）仍不做。

### 8.7 实现锁定决策（2026-06-29）—— 待 implement，按此执行

- **②预览一致性＝β 严格一致**：渲染做成**单一共享模块**（PixiJS/TS，自包含）；游戏 HUD 与编辑器预览（QtWebEngine 嵌同款 JS）跑**同一份代码** → 像素级一致、零"画两套"漂移。
- **纯程序化·零美术**：软圆 puff 贴图**代码烘一次**（离屏 canvas 径向渐变 / PixiJS v8 `FillGradient`→RenderTexture），**非 PNG、零美术**；气缕＝复用该贴图的 tinted `Sprite` 池（~30–40 个），香粉发亮＝`blendMode:'add'`；PixiJS 批处理 ≈1 draw call，**开销近零、60fps 随便跑**。mockup 的 canvas2D 每帧 createRadialGradient 只是原型画法，实机不照搬。
- **③换味过渡＝交叉淡入淡出**（A 淡出 + B 淡入、各保各参，自然不突兀，不出"半尸臭半香火"怪物）；同味变强弱/方位＝lerp。过渡时长/曲线做成 profile 可配参。
- **④参数机制＝命名预设库（profile 即预设）+ 参数曲线**（如香粉亮度包络）。
- **香粉味特殊（定稿）**：自体发亮（`add`＋冷色 bloom 光晕）＋**亮度包络＝陡攻击 → 顶部停留 → 长尾慢淡**（攻击/停留/衰减/峰值 四参逐味可配）＋盘卷不散＋几缕逆着往下够（呼应"盲目不撒手"）＋基线被照亮发抖。**只在濒死盲目显形那几拍出现**，守冷框架（§二/§六）；普通味永远哑光安静。
- **SmellSystem 扩持续状态**：加 `dir`（方位）/`flicker`（稳·动）/envelope；`setSmell(scent, intensity, [dir], [flicker])`；`clearSmell`＝**回落正常态（基线仍在）**。
- **嗅 action**：新增 `sniff`（add-game-action 三件套：运行时 register + 编辑器 ACTION_TYPES/schema + validator）+ 绑一个"嗅"键，按一下当前气缕短暂变浓变清（强度临时拉高、方位更明确）。
- **数据驱动**：`SCENTS` 从 `HUD.ts` 硬编码搬成 `smell_profiles.json`（schema＝基础飘法参数 + 可选"特殊渲染块"如香粉发亮/包络 + 过渡参数）；HUD/预览都从数据渲染。
- **收尾验证**：`tsc --noEmit` + `./dev.sh validate-data` + 素材审计 `--strict` + preview 实拍（驱动各味/各属性/香粉包络各截一张看对错）。

## 九、数据驱动落地设想（设计向，非实装）

照项目数据驱动原则，气味应尽量做成**数据条目 + 通用机制**：
- **气味是数据**：每种气味一条定义（id / 名 / 调性色 / 默认强度 / 用途标签）。
- **场景挂气味源**：场景/热区/实体上挂"气味源"（类型 + 强度 + 半径），由系统聚合出"当前主导气味"。
- **嗅 action**：新增 `sniff`（或并入交互），输出此处气味读数。
- **阈值/触发走 config**：灵敏度、阳气耦合系数等进 `game_config`。
- **香粉味**：作为一个**系统级气味源**，由死亡系绳/濒死时刻投放（和现有 `signal_cues` 对齐），不写进普通场景气味源。
- **和现有接口**：气味读数可作为**条件叶子**喂知识系统/规矩判定（若要，需评估是否新增条件类型=L2）。

> **范围标注**：整套是 **L3 新系统**（新感官 channel + 新 action + 新 HUD 元件 + 数据结构），**不是一遍数据落地能接进去的**。本文档只定设计，实装按 feature-iteration 分期。

## 十、MVP 范围与分期

**别一口吃。** 先验证"好不好玩、立不立得住"，再扩。

- **MVP**：嗅 action + HUD 气味丝 + **3–4 种有实际用途的味**（尸臭"该臭不臭" / 阴腥方位 / 香火 / ★香粉味）+ 香粉味接死亡系绳那一处。够验证"系统让香粉味可读"这个核心命题。
  - **实装进度（2026-06-29 修订）**：表现方案定稿为 **§八·方案 E（双层·基线+浮现）**；v1 范围＝核心编码（色/飘法/浓淡/沉浮/香粉冷读）＋ 逼近·波动·方位·嗅 action 全做。旧版 appear/disappear 气味烟（`HUD.ts`）**需按 §八 替换重做**；`SmellSystem` 持续状态扩 `dir`/`flicker`；`setSmell` 扩参；新增 `sniff` 输入/命令。待按 §八 实装。
- **二期**：扩气味种类、接知识系统线索、接规矩/遭遇钩子、阳气耦合灵敏度。
- **三期**：状态耦合（酒浸血盖味等）、更细的方位/层次表现。

## 十一、风险与纪律（写死，防翻车）

1. **别做成第二根被忽略的条**：每种气味**必须有玩法用途**，否则就是带 HUD 的 flavor，和现在的香粉味一样废。
2. **守冷框架**：系统让香粉味**可读**、不是**常见**；香粉味/小调仍只在濒死盲目显形、是阿秀专属，环境味绝不借用。
3. **不上全屏、不喊注意**：气味是冥冥之中、是工具，不是 alert（同三把火纪律）。
4. **只演现象、不出机理词**：给玩家只演气味与打架的说法，"气/场/耦合/读气"等一律幕后（煞机理两层铁律）。
5. **别过度建**：L3 大系统，按 MVP 分期；没验证好玩前不铺全套。
