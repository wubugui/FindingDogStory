# 阅读引导与奖励系统设计文档

> 不强迫玩家读文案，靠好奇心和实用需求驱动。遭遇面板是「考试」，规矩本是「课本」。
> 上级文档：《遭遇系统总设计文档》

---

## 一、设计原则

1. **永远不强迫阅读。** 不弹窗挡路，不设阅读前置条件。
2. **制造阅读动机。** 让玩家带着问题去翻规矩本，而非无目的地通读。
3. **奖励是内生的。** 读了之后你变聪明了，变聪明意味着消耗更少走得更远。不发「阅读奖励铜钱」。
4. **规矩本越用越有价值。** 不是静态的信息手册，而是会随玩家经历动态更新的活文档。

---

## 二、阅读动机的三个来源

### 2.1 吃亏引导（最主要的动机来源）

即时遭遇失败后的结果文本植入线索钩子：

**设计规则**：
- 每条失败文本必须包含至少一个指向规矩的线索
- 线索不给答案，只给方向
- 用关二狗的内心独白，不用系统提示

**文本模板示例**：

| 遭遇类型 | 失败文本中的线索钩子 |
|---------|------------------|
| 鬼类 | 「那东西似乎怕什么......你好像在哪里听人说过类似的事。」 |
| 鬼类（用错道具） | 「桃木片扔过去毫无反应。也许......不该扔，该怎么用才对？」 |
| 僵尸类 | 「那股腐臭味快把你熏晕了。老道士好像说过，碰到这种味道该怎么办来着......」 |
| 祟类 | 「头疼欲裂，东西又丢了。这到底是自己倒霉，还是碰上什么了？」 |
| 煞类 | 「你感觉被什么看不见的东西压着，喘不上气。跑也没用，这不是一个「东西」。」 |
| 假警报误判 | 「白费了一张符纸。要是能分辨真假就好了......」 |

### 2.2 NPC 反馈（社交层引导）

玩家从危险区狼狈回来后，NPC 可感知并主动提供信息：

**触发条件**：玩家身上有「撞邪」debuff 或「惊魂未定」状态时，与特定NPC对话会触发额外对话分支。

**NPC反馈示例**：

| NPC | 触发条件 | 反馈内容 |
|-----|---------|---------|
| 老辫子 | 玩家有「撞邪」 | 「你小子脸色不对啊。是不是碰上什么了？我跟你说，碰上这种事得......」→ 提供一条相关规矩的象层/碎片 |
| 茶馆老板 | 玩家从某区域撤退 | 「你去了XX？那地方最近不太平啊。前几天还有人说......」→ 提供环境情报 |
| 老道士 | 玩家有「撞邪」 | 可以花铜钱让老道士「看看」→ 直接告诉你遇到的是什么 + 怎么应对（相当于买到理层知识） |

### 2.3 好奇心驱动（自发探索）

规矩本本身的设计应该让翻阅变得有趣：

- 碎片文本有出处（「说书人张叨叨」「赶尸匠手札」），像真的在拼凑信息
- 不同碎片之间偶有矛盾（某些传闻互相冲突），激发玩家思考
- 规矩的「验证状态」标签让玩家想去验证（「这条规矩到底管不管用？」）

## 三、遭遇笔记系统

### 3.1 概述

每次即时遭遇结束后（无论成功失败），规矩本中自动添加一条「遭遇笔记」。

### 3.2 笔记内容

笔记以 **关二狗第一人称** 书写，包含：

- **遭遇描述**：在哪里碰到的、看到了什么信号
- **应对记录**：用了什么、结果如何
- **关二狗的思考**：他对这次经历的理解或困惑

### 3.3 笔记示例

**成功应对**：
> 城隍庙后山，半夜。油灯灭了，脖子后面一阵发凉。我掏出桃木片点了火，那股凉意就散了。看来桃木火确实管用，老道说的没错。

**失败应对**：
> 义冢那边，灯突然灭了。我扔了张符纸出去，啥反应都没有，白瞎了。后来跑出来的，丢了几文钱。那到底是什么东西？符纸不管用的话......得想想别的办法。

**假警报误判**：
> 后山小路上，听到沙沙声，吓得我扔了一片桃木。结果是只野猫。妈的，白瞎了。下次得先搞清楚再动手。

### 3.4 笔记与规矩的关联

遭遇笔记可以和规矩条目交叉引用：
- 如果遭遇涉及的实体类型有对应的规矩，笔记中自动标注关联（「这可能和XX规矩有关」）
- 玩家在翻看某条规矩时，如果有相关的遭遇笔记，会在规矩页面下方显示关联笔记

## 四、规矩交叉发现系统

### 4.1 概述

当玩家拥有两条及以上相互关联的规矩时，系统自动在规矩本中生成一条「关联发现」笔记。

### 4.2 触发条件

关联关系在数据中预定义：

```
关联定义 {
  ruleIdA: 第一条规矩
  ruleIdB: 第二条规矩
  requiredLayerA: A需要解锁到哪一层才触发（至少象层）
  requiredLayerB: B需要解锁到哪一层才触发
  discoveryText: 关联发现的文本
}
```

### 4.3 示例

| 规矩A | 规矩B | 关联发现文本 |
|-------|-------|-----------|
| 白毛僵尸怕火 | 桃木属阳 | 「桃木怕火，桃木又属阳......桃木点火是不是对付僵尸的好法子？」 |
| 城隍庙三更勿去 | 唱戏白影的来历 | 「后山的白影子会唱戏......柳家戏班子的人。她不是害人，她是在找人。那三更不去......是怕撞上她？还是怕打扰她？」 |
| 净堂判死因 | 驱鬼术 | 「净堂屎能判断死因......那如果是横死的，怨气更重，驱鬼的法子是不是得不一样？」 |

### 4.4 关联发现的作用

- **暗示组合用法**：提示玩家两条规矩可以配合使用
- **深化世界理解**：让玩家感觉自己在「想通」一些事情
- **引导阅读**：玩家看到关联发现后可能会去重新翻看两条规矩的详细文本

## 五、阅读奖励体系

所有奖励都是内生的——读了之后你在遭遇中的表现变好了。

| 奖励类型 | 触发时机 | 具体效果 |
|---------|---------|---------|
| 碎片集齐 → 规矩合成 | 集齐某条规矩的所有碎片 | 合成完整规矩，解锁该规矩在遭遇中的全部对应选项 |
| 象层解锁 → 识别标注 | 获得某类实体的辨识规矩象层 | 以后遭遇该类实体时，UI自动标注类型（不再显示「？」） |
| 理层解锁 → 快层增效 | 获得某条规矩的理层 | 快层中使用对应道具的成功率被动提升 |
| 术层解锁 → 解锁预制配方 | 获得某条规矩的术层 | 可在安全区制作该术对应的复合道具 |
| 术层解锁 → 解锁血/尿术 | 获得某条规矩的术层 | 遭遇中出现消耗血/尿的确定性应对选项 |
| 遭遇笔记积累 | 每次遭遇后自动 | 规矩本变得更丰富，关联更多 |
| 交叉发现 | 拥有两条关联规矩 | 暗示新的应对组合，提升理解深度 |
| 规矩验证 | 成功使用某条「未验证」规矩 | 规矩标记变为「有效」，增加玩家对该规矩的信心 |

**关键**：这些奖励不是「读完文案弹出一个+10铜钱」。它们是无声的、系统性的能力提升——下一次遭遇时，你会发现选项变多了、成功率变高了、消耗变少了。这才是阅读的回报。

## 六、玩家成长曲线

```
初期（什么都不懂）:
  遭遇 → 乱扔道具/逃跑 → 频繁失败 → 资源消耗大 → 很快被迫撤退
  失败文本给线索 → 回安全区翻规矩本 → 「原来如此」
  NPC主动提供信息 → 获得第一批象层规矩

中期（有了基础知识）:
  遭遇 → 识别类型 → 快层精准应对常见类型 → 偶尔失败
  碰到新类型 → 开面板仔细看 → 学到新知识
  交叉发现开始出现 → 规矩本越来越有用

后期（老手）:
  大部分遭遇快层秒解 → 很少开面板
  只有罕见高级遭遇才需要认真对待
  规矩本已经是一本厚厚的个人手册，充满遭遇笔记和关联发现
```

## 七、与其他系统的交互

| 系统 | 交互方式 |
|------|---------|
| 即时遭遇系统 | 提供失败文本中的线索钩子模板；遭遇结束后触发笔记写入 |
| 规矩系统 | 笔记和交叉发现关联到具体规矩条目；规矩验证状态更新 |
| 档案系统 | 遭遇笔记和交叉发现作为规矩本的动态子内容 |
| NPC对话系统 | debuff状态触发NPC额外对话分支 |
| Flag系统 | 遭遇经历写入flag，供NPC对话和后续判断使用 |
