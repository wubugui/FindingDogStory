## 一、实现总则（规范对齐声明）

1. 本关是**双幕因果关**：去程的玩家选择写入 Flag，回程的状态演出按 Flag 取变体。**因果闭环在数据上的形态就是：去程写 flag，回程读 flag**
2. 严格遵守规范边界：
   - Transition 只表达迁移与触发来源，不承载 Action
   - 演出由状态 onEnter 启动；Zone/Hotspot 的响应只发叙事事件（"状态拥有演出"）
   - **选择属于 DialogueGraph**（玩家主动交互后的响应）：X点②③的喊名选择走 DialogueGraph 的 choice + runActions；**选择的后果（回程世界变化）不走 DialogueGraph**，由回程状态 onEnter 按 flag 落地
   - 条件取变体（按 flag 选 Cutscene）由**底层 condition 系统**承担（规范：旧系统负责落地）
3. **双保底在图上的形态**：
   - 点①半脚本 → `x1_scripted` 是**必经状态**，`callname.x1 = right` 是**无条件写入**——保底账在图结构上不可绕过
   - 层4层5必触发 → `climax` 是 9 种选择组合的**唯一汇流点**，三条回程点位状态全部单向汇入
4. 失败不卡关：`answered` 分支无死端，经 `escape` 链汇回 `descend`

### 3.1 接活

```text
State dormant: 未开始
```

### 3.2 去程（舞台递进：孤立感/天色/闲扯狠度三线同步）

```text
State outbound_leg0: 出发 → 点①
  onEnter:
    switchScene(scn_mountain_road)
    setSceneLighting(daylight)             ← 规范缺口 #1
    playScriptedDialogue(banter_leg0)      （脚帮打趣：鬼鬼祟祟干啥去/工钱闲账）
    setQuestState(quest_yanwang_delivery, active)
  transition:
    callname.x1_reached -> x1_scripted
    （事件来源：zone_x1 进入）

State x1_scripted: X点① 半脚本（必经）
  onEnter:
    startCutscene(cs_x1_scripted)
    （演出接管：离队→走到林边→他清清嗓子顺口喊出→林中极轻一"应"→
      他愣半息又笑自己多心→追上大队，没人注意他离开过。
      演出内 runActions：setFlag(callname.x1 = right)）
  transition:
    cs_x1_scripted.complete -> outbound_leg1
  ⚠ 本状态无选择、无旁路。保底账在此无条件挂上。

State outbound_leg1: 闲扯① · 癫子周
  onEnter:
    setSceneLighting(dusk_begin)           （擦黑）
    playScriptedDialogue(banter_leg1)      （螺旋酒楼，半真半笑，还有人哄笑）
  transition:
    callname.x2_reached -> x2_choice
    （事件来源：zone_x2 进入）

State x2_choice: X点② 开放选择
  onEnter:
    showHint(离队走到林边)
    （hotspot_x2_edge 经因果链进入 armed；zone_x2_pass 同时 armed，见 4.5/4.6）
  transition:
    callname.x2_resolved -> x2_feedback
    （事件来源：dg_x2_choice 的 runActions，或 zone_x2_pass 越线自动结算"不喊"）

State x2_feedback: X点② 即时反馈
  onEnter:
    startCutsceneByCondition:              ← 条件取变体，底层 condition 承担
      callname.x2 == right   -> cs_x2_fb_right （回应慢半拍、音不太对；他皱眉回头）
      callname.x2 == wrong   -> cs_x2_fb_wrong （窸窣更杂、有什么在挪动；后脖子凉）
      callname.x2 == skipped -> cs_x2_fb_skip  （林子太静，静得他加快脚步）
  transition:
    cs_x2_fb_*.complete -> outbound_leg2

State outbound_leg2: 闲扯② · 马屠户（没人接茬）
  onEnter:
    setSceneLighting(dusk_deep)            （最暗的一段配最狠的故事）
    playScriptedDialogue(banter_leg2)
  transition:
    callname.x3_reached -> x3_choice

State x3_choice: X点③ 开放选择
  onEnter:
    showHint(离队走到林边)
    （hotspot_x3_edge armed；zone_x3_pass armed。点③在沟里，完全听不见人声）
  transition:
    callname.x3_resolved -> x3_feedback

State x3_feedback: X点③ 即时反馈
  onEnter:
    startCutsceneByCondition:
      callname.x3 == right   -> cs_x3_fb_right （像有什么"醒了"：鸟惊飞、枝叶无风自动）
      callname.x3 == wrong   -> cs_x3_fb_wrong （乱成一团、像捅了窝；他撒腿追大队）
      callname.x3 == skipped -> cs_x3_fb_skip  （几乎小跑着离开；归队时大队已走远一截）
  transition:
    cs_x3_fb_*.complete -> outbound_leg3

State outbound_leg3: 闲扯③ · 下蛊寨子（说一半住口）
  onEnter:
    setSceneLighting(dark)
    playScriptedDialogue(banter_leg3)      （说到一半自己住了口；之后无人说话）
  transition:
    callname.ridge_reached -> delivery
    （事件来源：zone_ridge 进入）
```

### 3.4 回程（一至三层强度 = 去程 flag；恍然 = 首个有声点）

```text
State return_p3: 回程点③
  onEnter:
    startCutsceneByCondition:
      callname.x3 == right   -> cs_ret_p3_right （清晰一声"二狗——"，认得他名）
      callname.x3 == wrong   -> cs_ret_p3_wrong （好几个声音抢着喊、急又杂——最凶）
      callname.x3 == skipped -> cs_ret_p3_quiet （只有风声，反常地静）
    若该点有声 且 !callname.awakening_done：
      前插 cs_awakening（恍然独白：那些名字是喊魂/是老子喊醒的/名还喊岔了）
      setFlag(callname.awakening_done = true)
    若该点有声 且 沉默玩法未启动：
      startMinigame(mg_silence_hold, phase = L2)   （拉扯30%，容错1-2次松手）
  transition:
    callname.return_p2_reached -> return_p2
    mg_silence_hold.fail -> answered

State return_p2: 回程点②
  onEnter:
    startCutsceneByCondition:（同上结构，按 callname.x2 取 right/wrong/quiet 变体）
    恍然与玩法启动判定同 return_p3（仍未触发则在此触发）
    若玩法已启动：setMinigamePhase(mg_silence_hold, L3)（拉扯60%，容错0-1次）
  transition:
    callname.return_p1_reached -> return_p1
    mg_silence_hold.fail -> answered

State return_p1: 回程点①（保底·人人必经的那一声）
  onEnter:
    startCutscene(cs_ret_p1_clear)
    （无条件版本：清晰一声"二狗——"。callname.x1 恒为 right，无变体）
    若 !callname.awakening_done：
      前插 cs_awakening（②③全没喊的玩家在此恍然：那是他自己亲口喊过的地方）
      setFlag(callname.awakening_done = true)
    若玩法未启动：startMinigame(mg_silence_hold, phase = L3)
    否则：setMinigamePhase(mg_silence_hold, L3)
  transition:
    callname.climax_reached -> climax
    mg_silence_hold.fail -> answered
  ⚠ 本状态保证：恍然这一拍与"必须按住"的体验，所有路径在此之前必然成立
```

### 3.5 层4层5（必触发汇流点）与双分支

```text
State climax: 层4 · 认不准（最危险）
  onEnter:
    startCutscene(cs_layer4_scrutiny)
    （喊声停→换呼吸声·吸气特别长·四面八方→视野边缘模糊→黑影缓慢打量→
      最后一声特别近、四面同时："二狗……"）
    setMinigamePhase(mg_silence_hold, L4)（拉扯95%，不容松手）
  transition:
    mg_silence_hold.complete -> release
    mg_silence_hold.fail -> answered

State release: 层5 · 散开（成功·Axiu 救场）
  onEnter:
    stopMinigame(mg_silence_hold)
    startCutscene(cs_axiu_release)
    （呼吸声与黑影像在犹豫→一缕极淡香粉味【vfx_axiu_signal + sfx_axiu_scent_proxy】→
      退散→风声夜鸟回归。两种读法都成立，不点破）
    setFlag(forest.answered = false)
  transition:
    cs_axiu_release.complete -> descend

State answered: 应声（失败分支入口）
  onEnter:
    stopMinigame(mg_silence_hold)
    startCutscene(cs_answered)
    （"哎！"→一切声音骤停·比安静更安静→屏幕白闪→脑子嗡白）
    setFlag(forest.answered = true)
  transition:
    cs_answered.complete -> escape

State escape: 逃亡
  onEnter:
    startMinigame(mg_escape_qte)（3-5 秒避障：树根/石头/崖边，保证能活）
  transition:
    mg_escape_qte.complete -> escape_out
  ⚠ mg_escape_qte 无 fail 出边——失败重试在小游戏内部消化，不产生叙事分支

State escape_out: 冲出林子
  onEnter:
    startCutscene(cs_escape_dawn)
    （冲出林子、天泛鱼肚白才停；反复呢喃"我应了……我应了……"）
  transition:
    cs_escape_dawn.complete -> descend
```

### 3.6 收尾

```text
State descend: 下岭收尾
  onEnter:
    startCutsceneByCondition:
      forest.answered == false -> cs_descend_held    （冷汗/疲惫/"别应……千万别应……"/灌酒含糊"山上风大"）
      forest.answered == true  -> cs_descend_answered（极度惊恐/呢喃"为什么要应……"/灌酒）
  transition:
    cs_descend_*.complete -> completed

State completed: 完成
  （镜像 flag：narrative.flow_forest_callname.completed.active
    ⑫ 终幕"回放林中那几声"读取本镜像；forest.answered 可供回放/外围文本取创伤深浅变体）
```


## 五、DialogueGraph 清单（只承担交互响应）

### dg_job_offer（接活 + 教名）

```text
节拍：跑腿活介绍 → 摊地图（三个X）→ 口头教名（三个名字的音频各播一次——玩家只"听"这一次）
     → 工钱规则（一个点几个铜板，喊完一起结）
choice:
  「接了」 runActions:
      sendNarrativeEvent(callname.job_accepted)
  「再想想」 退出对话（npc 停留在 available，可再接）
纪律：雇主不解释为什么喊、名字是谁——问多了钱就没了，对话里也不给问的选项
```

### dg_x2_choice / dg_x3_choice（相近名选择）

```text
进入即面对黑沉沉的林子。choice 四项（顺序随机）：
  名X（正确）        runActions: setFlag(callname.x2 = right)   + sendNarrativeEvent(callname.x2_resolved)
  名X′（读音相近·错） runActions: setFlag(callname.x2 = wrong)   + sendNarrativeEvent(callname.x2_resolved)
  名X″（读音相近·错） runActions: setFlag(callname.x2 = wrong)   + sendNarrativeEvent(callname.x2_resolved)
  「算了，不喊」      runActions: setFlag(callname.x2 = skipped) + sendNarrativeEvent(callname.x2_resolved)
纪律：
  - 喊出去的演出不在对话里——由 x2_feedback 状态 onEnter 按 flag 启动（状态拥有演出）
  - 选项文案不标对错；正确名只在 dg_job_offer 的音频里出现过那一次
  - dg_x3_choice 同构，flag 与事件换为 x3
```

## 六、Flag 与镜像状态

```text
手写 flag（经 runActions / 状态 onEnter 写入）：
  callname.x1            恒 = right（cs_x1_scripted 内无条件写入）
  callname.x2            ∈ { right, wrong, skipped }
  callname.x3            ∈ { right, wrong, skipped }
```

## 十、规范缺口 / 引擎需求清单

> 本关需要、但规范第一阶段未覆盖的点。实现前需引擎侧确认。

1. **`setSceneLighting(相位)`**：去程天色三段递进（daylight → dusk_begin → dusk_deep → dark）。规范 Adapter 只有 NPC/Hotspot/Zone，无场景级补丁。备选：每 leg 切不同光照变体场景（资产翻倍，**不建议**）；建议新增 Scene 级 Action 或第四类 SceneAdapter
2. **`setMinigamePhase` / 运行中参数更新**：mg_silence_hold 需要在不中断长按的前提下切 L2→L3→L4 相位。若 MinigameManager 不支持运行中参数，降级方案：分相位多实例（mg_hold_L2/L3/L4 各自 start，complete 后接续）——代价是相位衔接处出现可松手的空窗，**需要引擎评估哪边成本低**
3. **`startCutsceneByCondition`**：按 flag 取 Cutscene 变体。底层已有 condition 系统，需确认 condition 可绑定在状态 onEnter 的 Action 上（规范 11.4 编辑器预览也依赖此能力）
4. **响应规则（事件→Action 的转写）**：`callname.x2_passed → setFlag + sendNarrativeEvent` 这类"事件触发一组 Action"的挂点，规范里最接近的是状态生命周期；若无独立响应规则机制，可把转写并入 zone_x2_pass.armed 的 onExit——需引擎确认惯用法
5. **闲扯的行进播报**：`playScriptedDialogue` 沿路异步播放（不打断行走）。规范 Action 表有"播放对白"，需确认支持非阻塞模式
6. **选项顺序随机**：dg_x2/x3_choice 的四个选项随机排序（防位置记忆），需确认 DialogueGraph choice 支持
