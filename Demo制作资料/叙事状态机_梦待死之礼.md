## 一、实现总则（规范对齐声明）

1. 本关是**纯线性演出关**：Flow 子图为单链，无分支、无失败边、无 Game Over
2. 严格遵守规范边界：
   - Transition 只表达迁移与触发来源，**不承载 Action**
   - 演出（Cutscene）一律由**状态的 onEnter** 启动；实体的 interactResponse 只做两件事：**发叙事事件 + 可选的即时轻反馈**（"状态拥有自己的演出"）
   - 老两口全程 `interactionEnabled = false`——他们是 Cutscene 演员，**不是可对话 NPC**（设计纪律：台词全部在演出里，玩家不能主动盘问）
   - DialogueGraph 本关**零使用**（无任何玩家主动对话）
3. **「不按不走」的实现就是状态机本身**：`meal_wait` 状态除 `dream.eat_pressed` 外**没有任何出边**——没有超时、没有旁白催促、没有兜底迁移。停表即演出
4. 入梦**不打任何"梦"标识**：本图所有 Cutscene 中不得出现"梦"字样字幕；唯一字幕是 `cs_wake` 开头的"两天后"

## 三、Flow 子图：flow_dream_rite

```text
Flow: flow_dream_rite

State dormant: 未开始
  transition:
    flow_zhuang_corpse.fall_into_pond.entered -> road
    （触发来源：② 背尸 Phase 6"**贪财捡帕子包 → 失足坠落深潭濒死**"状态进入；跨子图因果，由 ② 背尸流程发出）

State road: 夜路
  onEnter:
    switchScene(scn_dream_road)
    fadeInFromBlack(slow)
    setPlayerMovement(forward_only)        ← 规范缺口 #1，见第九节
    playAmbient(amb_dream_silence)         ← 只有脚步声的"安静"，见第九节 #5
  transition:
    dream.reach_house -> threshold
    （事件来源：zone_dream_house 进入）

State threshold: 门口（门先开了）
  onEnter:
    startCutscene(cs_door_open)
  transition:
    cs_door_open.complete -> meal_wait

State meal_wait: 吃饭·停表
  onEnter:
    showSinglePrompt(【吃】)               ← 规范缺口 #2，见第九节
  transition:
    dream.eat_pressed -> meal_eat
  ⚠ 本状态无其他出边。不按，画面不走。

State meal_eat: 吃饭·演出
  onEnter:
    hideSinglePrompt()
    startCutscene(cs_eat)
  transition:
    cs_eat.complete -> clothes_wait

State clothes_wait: 换衣·等待
  transition:
    dream.touch_clothes -> clothes_change
    （事件来源：hotspot_shroud 交互）

State clothes_change: 换衣·演出
  onEnter:
    startCutscene(cs_clothes)
  transition:
    cs_clothes.complete -> bed_intro

State bed_intro: 引路与烧纸
  onEnter:
    startCutscene(cs_bed_intro)
  transition:
    cs_bed_intro.complete -> bed_wait

State bed_wait: 门板前·等待
  transition:
    dream.lie_down -> rite
    （事件来源：hotspot_doorboard 交互）

State rite: 收束（喊魂→盖纸→纸停→小调→纸颤）
  onEnter:
    startCutscene(cs_rite)
  transition:
    cs_rite.complete -> wake

State wake: 两天后
  onEnter:
    switchScene(scn_country_road_day)
    startCutscene(cs_wake)
  transition:
    cs_wake.complete -> completed

State completed: 完成
  （镜像 flag 自动生成：narrative.flow_dream_rite.completed.active
    供 ③ 茶馆流程作为进入条件读取）
```

**Flow 编排视图（一条线）**：

```text
dormant → road → threshold → meal_wait → meal_eat
→ clothes_wait → clothes_change → bed_intro → bed_wait
→ rite → wake → completed
```

## 九、规范缺口 / 引擎需求清单

> 以下是本关需要、但规范第一阶段（NPC/Hotspot/Zone 三类 Adapter + 既有 Action 表）未覆盖的点。**实现前需引擎侧确认**，确认前本文不擅自发明新概念。

1. **`setPlayerMovement(forward_only)`**：夜路"只能朝那户人家走"。规范 Action 表无玩家移动限制类 Action。备选降级：把夜路做成可打断节奏的长 Cutscene（牺牲"自己走过去"的代入，**不建议**）
2. **`showSinglePrompt(【吃】)`**：全屏唯一按键的 UI 呈现。属表现层，不影响状态图结构；若无此 UI 能力，降级为普通 Hotspot 高亮（牺牲"屏幕上只有一个键"的压迫感）
3. **跨子图触发来源**：`flow_zhuang_corpse.escape_hide.entered` 要求 ② 背尸的流程图先按同规范立图；若 ② 暂未立图，临时入口可用旧系统 `sendNarrativeEvent(dream.enter)`（规范 14.3 新增 Action 类型，已支持）
4. **Cutscene 完成事件**：规范事件来源表已含"Cutscene 完成"，按 `cs_*.complete` 命名约定即可
5. **`playAmbient`**：环境底噪播放。若 Action 表无此项，并入 switchScene 的场景配置
