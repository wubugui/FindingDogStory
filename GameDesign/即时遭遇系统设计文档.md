# 即时遭遇系统设计文档

> 本文档详细定义即时遭遇的操作模式、触发机制、选项生成逻辑和数据结构。
> 上级文档：《遭遇系统总设计文档》

---

## 一、定位

即时遭遇是游戏的「杂兵战斗」等价物。它不讲故事，它制造压力。

- **体量**：10-30秒决策窗口
- **频率**：在危险区域内按概率随机触发
- **核心体验**：出门前策略（带什么）+ 当场快速反应（用什么）
- **作用**：消耗资源、维持危险感、提供小型奖励、训练玩家熟悉规矩

---

## 二、触发机制

### 2.1 触发条件

即时遭遇在 **危险区域** 内根据以下条件判定是否触发：

| 条件 | 说明 |
|------|------|
| 区域 | 当前场景的区域标记为危险 |
| 时间 | 可选，某些遭遇仅在特定时间段触发 |
| Flag条件 | 可选，依赖全局标记状态（主线进度、支线状态等） |
| 触发概率 | 每次进入新的区域检测点时按概率判定 |
| 冷却 | 同一模板遭遇在短时间内不重复触发 |
| 排斥 | 当前正在进行支线解谜链的特殊阶段时，不触发即时遭遇 |

### 2.2 触发时机

玩家在危险区域内 **移动经过特定检测点** 时判定触发。检测点在场景数据中定义（可以是热区边界、路径节点等），不是每帧判定。

### 2.3 假警报

遭遇池中包含一定比例的假警报（20-30%）。假警报表现出异常信号但没有实际威胁。

- 如果玩家对假警报使用道具 → 道具浪费
- 如果玩家无视假警报 → 无后果
- 有对应象层知识的玩家可以更准确地识别假警报

假警报的作用：让「要不要应对」本身成为一个决策，避免玩家对每个信号都条件反射式地使用道具。

## 三、双层操作模式

### 3.1 快层（快捷键即时反应）

#### 操作流程

1. 异常信号出现（画面/音效变化，1-3秒预告）
2. 如果有对应象层 → UI角落闪过实体类型标识；否则显示「？」
3. 玩家按快捷键使用预绑定的道具
4. 系统根据道具与实体类型的克制关系判定结果
5. 结果反馈（短文本+音效），继续探索

#### 快捷栏设计

- 玩家在安全区可以将道具绑定到 **3-4个快捷键**
- 快捷栏内容在进入危险区后锁定，不可中途更换
- 常见配置示例：
  - 1号键：桃木片（对鬼类通用）
  - 2号键：糯米（对僵尸类）
  - 3号键：朱砂（万能但贵）
  - 4号键：烧酒（临时驱散）

#### 快层中规矩的作用

快层不打开规矩面板，但规矩知识 **被动** 影响快层效果：

| 知识状态 | 快层表现 |
|---------|---------|
| 无知识 | 道具选项盲用，成功率低，可能浪费 |
| 有象层 | UI标注实体类型，玩家知道该按哪个键 |
| 有理层 | 对应道具的成功率被动提升（同一个桃木片，有理层比没理层效果好） |
| 有术层 | 快层中额外出现「血术/尿术」快捷选项（如果提前预绑定了） |

### 3.2 慢层（规矩面板策略模式）

#### 开启条件

- 遭遇发生后，玩家可选择开启规矩面板
- **需要消耗特定道具或满足特定条件**（具体道具/条件待定）
- 有些突发遭遇（太近/太突然）不允许开面板，只能用快层

#### 面板内容

面板打开后游戏暂停。面板展示：

**观察区域**：
- 当前遭遇的异常信号描述
- 如果有象层知识 → 显示实体类型判定和已知特征
- 如果没有 → 只显示原始信号，玩家需要自行推断

**判断区域**：
- 列出玩家已掌握的、与当前遭遇可能相关的规矩
- 有理层的规矩标注推荐度（基于与当前实体的匹配）
- 无理层的规矩只显示象层描述，玩家需要自行判断是否适用

**行动区域**：
- 根据携带道具+规矩知识生成可执行的应对选项
- 每个选项标注：消耗什么、预估效果（基于知识水平）
- 术层选项标注血/尿消耗量
- 行为规矩选项标注执行要求（如「接下来5秒不按反方向」）

#### 面板不限时

面板打开后不限时。设计理由：
- 规矩文案是世界观的重要载体，需要让玩家有时间阅读
- 限时会让玩家跳过文案，损害沉浸感
- 面板开启本身有资源代价，这已经构成了使用限制
- 简单遭遇玩家用快层解决，不会开面板；只有棘手的才开，此时就应该给足思考时间

### 3.3 不开面板也不用快层的情况

如果玩家既不按快捷键也不开面板，有一个 **超时窗口**（10-20秒）。超时后视为「硬扛」，按遭遇的默认失败结果处理（通常是轻度后果）。

## 四、选项生成逻辑

即时遭遇的选项不是预先写死的，而是根据玩家当前状态动态生成的。

### 4.1 快层选项生成

```
输入：玩家快捷栏绑定的道具 + 遭遇的实体类型 + 玩家规矩知识
输出：每个快捷键是否可用、预估效果

对于快捷栏中的每个道具：
  if 道具在该实体类型的「有效道具」列表中:
    if 玩家有该实体的理层知识:
      成功率 = 高
    else:
      成功率 = 中等
  elif 道具在「微效」列表中:
    成功率 = 低
  else:
    成功率 = 极低（浪费）

额外选项（术层）:
  if 玩家有对应术层 + 血量足够 + 预绑定了「血术」:
    生成血术快捷选项，成功率 = 确定
  if 玩家有对应术层 + 尿量足够 + 预绑定了「尿术」:
    生成尿术快捷选项，成功率 = 高（对低级有效）

永远存在:
  [逃跑] - 无消耗，小惩罚
```

### 4.2 慢层（面板）选项生成

```
输入：玩家背包全部道具 + 全部已学规矩 + 遭遇实体类型 + 身体资源状态
输出：完整的应对选项列表

基础选项（永远存在）:
  [逃跑] - 小惩罚
  [硬扛] - 承受默认后果

道具选项（根据背包中所有相关道具生成）:
  对于背包中每个可用道具:
    生成使用选项，标注消耗量和预估成功率
    如果有理层知识：成功率标注提升

术层选项（根据术层规矩+身体资源生成）:
  对于每条已学术层:
    if 与当前实体类型匹配:
      生成血术选项，标注血量消耗和效果
      if 可用尿替代:
        生成尿术选项，标注尿量消耗和效果

行为规矩选项（根据已学行为类规矩生成）:
  对于每条行为规矩:
    if 与当前实体类型匹配:
      生成行为选项，标注执行要求（如「5秒不回头」）

预制复合道具选项:
  if 背包中有预制品且匹配当前实体:
    生成使用选项，标注效果（通常是确定成功）
```

### 4.3 成功率体系

| 应对方式 | 基础成功率 | 有理层加成 |
|---------|----------|----------|
| 随机扔道具（无象层） | 20-30% | -- |
| 针对性道具（有象层） | 50-60% | 提升到 75-85% |
| 术层（血） | 90-100% | -- |
| 术层（尿） | 70-80%（对低级） | -- |
| 预制复合道具 | 95-100% | -- |
| 行为规矩 | 取决于执行（操作成功则100%） | -- |
| 逃跑 | 80%脱身 | -- |

## 五、遭遇的参数化模板

### 5.1 实体类型引用

每个即时遭遇模板引用一个实体类型（定义在《实体图鉴设计文档》中），实体类型决定了：
- 可能出现的信号（真信号+干扰信号）
- 有效的道具/规矩/行为
- 危险等级

### 5.2 模板数据结构

```
即时遭遇模板 {
  id: 唯一标识
  entityTypeId: 引用的实体类型
  
  触发条件: {
    areas: 可触发的场景列表
    time: "day" | "night" | "any"
    flags: Flag条件数组
    probability: 触发概率（0-1）
    cooldown: 冷却时间
  }
  
  危险等级: 1 | 2 | 3
  
  信号配置: {
    realCount: 展示几个真信号（从实体类型的信号池中随机抽取）
    noiseCount: 混入几个干扰信号
  }
  
  奖励（成功时）: 动作列表（给铜钱、给碎片、设flag等）
  惩罚（失败时）: 动作列表（扣资源、加debuff等）
  
  叙事文本: {
    发现: 异常出现时的描述文本（数组，随机选一条）
    成功: 应对成功的描述
    失败: 应对失败的描述 + 线索钩子
    假警报揭示: 如果是假警报，揭示真相的描述
  }
}
```

### 5.3 假警报模板

```
假警报模板 {
  id: 唯一标识
  signals: 假信号组合
  triggerConditions: 同上
  revealText: 揭示真相的描述（「原来是穿堂风」）
  wastedItemRefund: 是否返还浪费的道具（通常不返还）
}
```

## 六、行为规矩的执行

在即时遭遇中选择行为规矩后，进入一个短暂的 **操作约束模式**：

### 6.1 行为规矩类型

| 类型 | 操作要求 | 持续时间 | 失败条件 |
|------|---------|---------|---------|
| 不回头 | 持续向前走，不按反方向 | 3-5秒 | 按了反方向键 |
| 不应声 | 不按任何交互键 | 3-5秒 | 按了交互键 |
| 不动 | 停止所有输入 | 3-5秒 | 按了任何移动/交互键 |
| 绕行 | 不走直线穿过某区域 | 到达安全点 | 进入禁止区域 |

### 6.2 干扰/诱惑

行为规矩执行期间，游戏会制造干扰诱使玩家违反约束：
- 身后传来脚步声/叫名声（诱使回头）
- 旁边出现交互提示（诱使按交互键）
- 地上出现掉落物（诱使停下捡东西）

干扰的强度随遭遇危险等级提升。

### 6.3 成功与失败

- 成功遵守约束 → 遭遇安全解决，无资源消耗
- 违反约束 → 视为遭遇失败，按正常失败后果处理

行为规矩是「免费但有风险」的选项：不消耗任何资源，但需要操作上的自制力。

## 七、与现有系统的集成

### 需要扩展的现有系统

| 系统 | 扩展内容 |
|------|---------|
| EncounterManager | 新增即时遭遇模式：支持快层/慢层双模式、超时默认失败、动态选项生成 |
| EncounterOptionDef | 新增字段：bloodCost、urineCost、bloodQualityRequired、behaviorRule、successRate |
| EncounterUI | 新增快层UI（快捷栏提示）和慢层面板UI（观察/判断/行动三区域布局） |
| RulesManager | 新增查询接口：根据实体类型返回匹配的规矩列表及其层级状态 |
| ZoneSystem | 区域数据新增即时遭遇配置（遭遇池引用、触发概率） |
| InputManager | 支持快捷栏按键绑定和行为规矩期间的输入约束判定 |
| HUD | 新增快捷栏显示、debuff层数指示器 |

### 新增系统

| 系统 | 职责 |
|------|------|
| TemplateEncounterManager | 即时遭遇的触发判定、模板实例化、信号随机组合、选项动态组装 |
| BehaviorRuleExecutor | 行为规矩的实时执行监控、干扰事件播放、成功/失败判定 |
| QuickSlotManager | 快捷栏道具绑定管理、进入危险区后锁定 |
