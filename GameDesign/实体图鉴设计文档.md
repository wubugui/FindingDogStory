# 实体图鉴设计文档

> 本文档定义游戏中所有可遭遇的超自然实体类型、其异常信号、克制关系以及与规矩系统的映射。
> 上级文档：《遭遇系统总设计文档》
> 底层依据：《超自然底层设定》（supernatural-lore.md）

---

## 一、设计原则

1. 实体分类基于《超自然底层设定》中的谱系，不另造新体系
2. 每种实体有独特的信号组合，不同类型之间信号可以重叠（增加辨识难度）
3. 克制关系基于民俗真实逻辑（桃木克鬼、糯米克僵尸等），不是任意设计
4. 每种实体至少对应一条规矩的象层（用于辨识）

---

## 二、实体大类

基于底层设定，游戏中实际可遭遇的实体分为以下大类：

| 大类 | 底层定义 | 游戏中的表现 | 危险等级范围 |
|------|---------|------------|------------|
| 鬼 | 人死后魂的残留，无形、阴性 | 精神影响、环境异象、不直接显形 | 1-3 |
| 僵 | 尸体因风水/怨气魄不散 | 有实体、有物理力量（间接暗示） | 2-3 |
| 妖 | 动物修炼成精，拟人化 | 以人形混迹，通过细节破绽辨识 | 1-3 |
| 祟 | 暗中作祟的邪气/小鬼 | 日常小异常：丢东西、头痛、运气差 | 1 |
| 煞 | 凶恶之气，无形无相 | 环境不对劲：温度、植物、水源异常 | 2-3 |
| 魑魅 | 山林川泽杂气，善迷惑 | 鬼打墙、听到叫名声、方向错乱 | 1-2 |

注意：玩家不会正面「看到」大部分实体。表现以间接环境异象为主。

---

## 三、异常信号库

异常信号是玩家在即时遭遇中感知到的线索。每个信号可能指向多种实体类型。

### 3.1 视觉信号

| 信号ID | 描述 | 可能指向 |
|--------|------|---------|
| sig_lamp_flicker | 油灯忽明忽暗 | 鬼、祟 |
| sig_shadow_wrong | 影子方向/形状不对 | 鬼 |
| sig_wet_footprint | 地面出现无源湿脚印 | 鬼 |
| sig_object_moved | 物品位置莫名变化 | 祟、精 |
| sig_plant_wilt | 周围植物异常枯黄 | 煞（阳煞） |
| sig_water_muddy | 水源突然浑浊 | 煞（阴煞） |
| sig_animal_flee | 动物异常回避/逃窜 | 煞、僵 |
| sig_npc_odd | NPC外貌/行为细节不对（不常见的反应、错误的习惯） | 妖 |
| sig_paper_ash_spin | 纸灰无风自旋 | 鬼 |
| sig_mirror_blur | 镜面短暂模糊 | 精（器灵） |

### 3.2 听觉信号

| 信号ID | 描述 | 可能指向 |
|--------|------|---------|
| sig_whisper | 模糊低语声 | 鬼 |
| sig_name_call | 远处有人叫你名字 | 魑魅 |
| sig_opera_sing | 隐约的戏曲唱腔 | 鬼（特定：柳家戏班地缚灵） |
| sig_scratch | 抓挠声/敲击声 | 僵、祟 |
| sig_silence | 突然的死寂（环境音消失） | 煞 |
| sig_wrong_sound | 不该有的声音（如室内水声） | 祟、鬼 |

### 3.3 体感信号

| 信号ID | 描述 | 可能指向 |
|--------|------|---------|
| sig_temp_drop | 周围骤然发冷 | 鬼、煞（阴煞） |
| sig_temp_rise | 周围异常闷热 | 煞（阳煞） |
| sig_nausea | 无来由的恶心/头晕 | 祟、煞 |
| sig_neck_cold | 后脖颈发凉 | 鬼 |
| sig_itch | 皮肤莫名发痒 | 蛊/虫（特殊） |

### 3.4 干扰信号（非超自然）

| 信号ID | 描述 | 真实原因 |
|--------|------|---------|
| sig_wind | 穿堂风 | 自然风 |
| sig_cat_cry | 猫叫 | 野猫 |
| sig_drunk_noise | 醉汉动静 | 喝多了的路人 |
| sig_rat | 老鼠窜过 | 老鼠 |
| sig_branch_crack | 树枝断裂声 | 自然朽坏 |

## 六、信号与实体的映射规则

即时遭遇触发时，系统从实体的真信号池中随机抽取 2-3 个，再从干扰信号池中混入 0-1 个，组成玩家看到的信号集合。

**辨识逻辑**：
- 玩家没有任何辨识规矩 → 只看到信号描述，显示「不明异常」
- 玩家有某类实体的象层辨识规矩 → 如果当前信号集合与该类实体的真信号匹配度超过阈值，自动标注「疑似XX」
- 多种实体共享部分信号 → 可能出现误判（信号指向鬼，但实际是祟）

**假警报处理**：
- 假警报使用干扰信号组合，没有真信号
- 有辨识规矩的玩家看到纯干扰信号时，不会出现实体类型标注 → 间接提示这可能是假的
- 没有规矩的玩家无法区分假警报和真遭遇

## 七、后续扩展

实体类型是数据驱动的，后续可通过新增 JSON 数据扩展：
- 新的实体类型（如特定支线中出现的独特实体）
- 新的信号（随区域和剧情扩展）
- 新的克制关系（随新规矩和新道具加入）

实体类型表也是支线解谜链的参考：支线中出现的超自然实体应该属于已定义的大类，遵循同样的信号和克制逻辑。
