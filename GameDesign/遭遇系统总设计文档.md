# 遭遇系统总设计文档

> 本文档是遭遇系统的顶层设计纲要。各子系统的详细规格见独立文档。

---

## 一、设计背景

### 为什么需要遭遇系统

本游戏没有战斗系统，但需要一个等价于战斗的核心机制来：
- 让危险区域真的危险（不是空走）
- 消耗玩家资源，驱动经济循环
- 提供可重复的核心玩法，撑起游玩时间
- 训练玩家使用规矩，为支线解谜链做铺垫

遭遇系统就是本游戏的「战斗系统」，只是操作语言不是攻防，而是判断、应对和遵守规矩。

### 设计原则

1. **志怪世界探索是第一目标。** 遭遇服务于世界体验，不是反过来。
2. **规矩（知识）是核心资产。** 玩家的成长不是等级变高，而是懂得更多、判断更准、消耗更少。
3. **身体是装备。** 关二狗没有盔甲武器，他的血和尿就是他的装备。
4. **策略在出门前，反应在遭遇时。** 进入危险区域之前的准备决定了遭遇时的表现。
5. **不强迫，靠内生动机。** 不强迫玩家阅读规矩文案，靠好奇心和实用需求驱动。

---

## 二、核心循环

```
学规矩 → 准备物资 → 进入危险区 → 遭遇消耗资源 → 成功/失败 → 回安全区补给学习
  ^                                                                    |
  └──────────────── 知识越多 → 消耗越少 → 走得越远 ────────────────────┘
```

循环的驱动力：**规矩（知识）的产出回流为下一轮的投入。**

- 学到规矩的象层 → 遭遇时能识别实体类型 → 减少误判和浪费
- 学到规矩的理层 → 同样的道具成功率提升 → 消耗更少
- 学到规矩的术层 → 解锁用血/尿执行确定性应对 → 效果最强
- 懂得更多 → 消耗更少 → 在危险区待得更久 → 接触到更深的区域 → 学到更深的规矩

这是一个自我强化的正循环，和「打怪→变强→打更强的怪」结构同构。

---

## 三、区域划分

用地图区域（而非昼夜）划分安全与危险：

- **安全区**（城镇、茶馆、码头等）：社交、经济、信息收集、学规矩、准备物资、恢复身体
- **危险区**（城隍庙后山、义冢、阎王岭等）：即时遭遇触发、支线解谜推进、资源消耗

危险区的遭遇池不是固定的，还受 **时间** 和 **逻辑条件（flag）** 影响：
- 同一个城隍庙，白天和夜晚遭遇不同
- 同一个区域，主线第一天和第五天（阴气扩散后）遭遇不同
- 完成某个支线后，某些区域的遭遇池会变化

这与现有的 FlagStore + 条件系统完全兼容。

## 四、两类遭遇

游戏中的遭遇分为两类，各司其职：

### 4.1 支线解谜链（大型内容）

已设计的替身纸人、假道士驱鬼、洋人背棺等。

| 特征 | 说明 |
|------|------|
| 体量 | 大型，15-60分钟 |
| 场景 | 跨多个地图，有完整叙事弧 |
| 触发 | 玩家主动接取 |
| 时间压力 | 无，可以回去准备再继续 |
| 规矩作用 | 三层是推进线索的钥匙 |
| 设计方式 | 手工编排，每条线独特 |
| 定位 | 游戏的内容骨架，展现志怪世界的深度和故事 |

沿用现有的任务+遭遇+对话系统，不需要重新设计。

### 4.2 即时遭遇（小型/短促）

本文档重点。等价于RPG中的杂兵战斗。

| 特征 | 说明 |
|------|------|
| 体量 | 小型，10-30秒决策 |
| 场景 | 当前危险区域内就地发生 |
| 触发 | 在危险区域自动随机触发 |
| 时间压力 | 有，当场决策 |
| 规矩作用 | 三层决定选项的质量和消耗 |
| 设计方式 | 参数化模板，由实体类型+信号+条件组合生成 |
| 定位 | 让危险区域持续危险，消耗资源，控制探索节奏 |

**即时遭遇不讲故事，它制造压力。** 故事由支线解谜链来讲。

### 4.3 两类遭遇的关系

- 即时遭遇是 **底层噪声**——维持危险感和资源压力
- 支线解谜链是 **信号峰值**——在噪声之上突出的完整体验
- 即时遭遇可以作为支线的 **入口**：处理一个看似普通的即时遭遇时，发现信号不寻常，被自然拉入支线
- 玩家不知道二者的边界在哪——这就是「光怪陆离」

## 五、规矩系统（象/理/术）与遭遇的映射

现有规矩系统的三层结构在遭遇中的具体作用：

### 5.1 在即时遭遇中

| 层级 | 作用 | 具体表现 |
|------|------|---------|
| 无知识 | 只能碰运气或逃跑 | 选项少、消耗大、成功率低 |
| 象（xiang） | 识别实体类型 | UI自动标注类型（而非「？」），解锁针对性道具选项 |
| 理（li） | 理解原理，精准选择 | 同样道具的成功率提升，减少浪费 |
| 术（shu） | 执行仪式/操作 | 解锁用血/尿执行的确定性应对选项，最强效果 |

### 5.2 在支线解谜链中

| 层级 | 作用 |
|------|------|
| 象 | 提供线索方向（「听说过这种东西」→ 知道去哪找答案） |
| 理 | 缩小排查范围（「知道它为什么」→ 排除错误方向） |
| 术 | 提供关键解法（「知道怎么做」→ 解谜的关键一步） |

### 5.3 举例：「白毛僵尸怕火」（rule_zombie_fire）

即时遭遇中碰到疑似僵尸：
- **无知识**：「扔东西试试」（随机消耗一个道具，可能没用）或「跑」
- **有象层**：识别出是白毛僵尸，「扔桃木片」选项出现（定向消耗，成功率中等）
- **有理层**：知道要点火而非扔，「点燃桃木」选项出现（同样消耗但成功率高）
- **有术层**：「血画引火符+点燃桃木」（消耗血量+桃木片，确定成功）

## 六、即时遭遇的双层操作模式

### 6.1 快层（快捷键即时反应）

- 出发前将常用道具绑定到快捷键
- 遭遇发生时不暂停，直接按快捷键使用
- 适合已熟悉的遭遇类型、低级遭遇
- 是「速刷」工具，用于应付杂兵、刷取小奖励

### 6.2 慢层（规矩面板策略模式）

- 遭遇发生时开启规矩面板，暂停游戏进入策略模式
- **开启需要消耗特定道具或满足特定条件**（具体待定），不是免费的
- 面板打开后 **不限时**，让玩家认真阅读规矩、理解世界、做决策
- 面板内按「观察→判断→行动」推荐流程做精确应对（玩家也可不严格遵守）
- 开启次数受资源限制：简单的用快层解决，棘手的才舍得开面板

### 6.3 快层与慢层的关系

| | 快层 | 慢层 |
|---|---|---|
| 暂停 | 不暂停 | 暂停 |
| 消耗 | 只消耗使用的道具 | 额外消耗开面板的道具 |
| 适用 | 已知/低级遭遇 | 未知/高难度遭遇 |
| 策略深度 | 低，纯反应 | 高，完整判断流程 |
| 玩家成长 | 新手少用，老手主用 | 新手常用，老手少用 |

有些突发遭遇不给开面板的机会（太突然/太近），只能用快层。

## 七、身体资源系统概述

关二狗的「装备」就是他自己的身体。

### 7.1 血（高价值资源）

- 消耗代价高（直接扣HP），效果强力（术层确定性解决）
- 恢复慢（吃饭/休息）
- 有血质属性（阳气浓度、童子血、酒浸血、药血），影响术层效果
- 管理核心：**取舍**——舍得放多少血

### 7.2 尿（时机管理资源）

- 消耗代价低（喝水即可补充），效果中等（驱散/防御/辅助）
- 恢复快（喝水，延迟生效）
- 膀胱有上限，满了不排有负面效果
- 管理核心：**时机**——什么时候排、什么时候存

### 7.3 血与尿的定位

血是「硬通货」，尿是「零钱」。两者互补不替代。

> 详细数值框架见《身体资源系统设计文档》。

## 八、预制复合道具

- 术层的复合操作在安全区提前制作（如：血+桃木片 → 引火符木）
- 制成品带进危险区，遭遇中直接使用
- 极少数高级术层规矩允许现场合成（稀缺的高级能力，消耗更大）

> 详细配方体系见《预制道具系统设计文档》。

## 九、失败累积与惩罚

### 设计原则

**单次失败不致命，连续失败迫使撤退。** 心疼但不绝望。

### 三档后果

| 档位 | 触发条件 | 后果 |
|------|---------|------|
| 轻度 | 低级遭遇失败 + 低debuff层数 | 丢少量资源 + 叠1层「不安」+ 线索钩子 |
| 中度 | 中级遭遇失败 或 debuff累积 | 丢较多资源 + 「撞邪」降血量上限 + 推出小区域 |
| 重度 | 高级遭遇失败 或 debuff达3层 | 强制撤退 + 随机丢道具 + 「惊魂未定」需休息恢复 |

成功应对遭遇减1层debuff。有准备的玩家能持续维持低debuff，在危险区待很久。

### 信息透明

玩家必须能预判风险：
- **区域危险等级**：进入前可感知（环境叙事：纸钱密度、NPC警告、氛围变化）
- **遭遇危险等级**：发生时可感知（异常信号的强度暗示危险程度）
- **自身debuff层数**：HUD上持续可见

> 详细计算公式和debuff效果见《失败累积与惩罚系统设计文档》。

## 十、阅读引导与规矩本奖励

### 设计原则

不强迫玩家读文案，靠吃亏后的内生动机驱动。遭遇面板是「考试」，规矩本是「课本」。

### 引导机制

1. **吃亏引导**：遭遇失败的结果文本植入线索钩子，不给答案
2. **NPC反馈**：玩家狼狈回来后，NPC感知并主动提供相关信息
3. **规矩本关联提示**：阅读某条规矩时，如果最近遭遇过相关实体，出现关联批注

### 阅读奖励（内生的）

| 奖励类型 | 机制 |
|---------|------|
| 碎片集齐 → 规矩合成 | 已有机制 |
| 象层 → 识别标注 | 遭遇时UI自动标注实体类型 |
| 理层 → 快层增效 | 对应道具成功率提升 |
| 术层 → 解锁预制配方 | 可在安全区制作复合道具 |
| 遭遇笔记 | 每次遭遇后自动记录关二狗的经历和思考 |
| 规矩交叉发现 | 拥有两条相关规矩时自动生成关联笔记 |

> 详细设计见《阅读引导与奖励系统设计文档》。

## 十一、子系统关系总览

```
┌─────────────┐
│   规矩系统    │ ← 核心知识资产
│  象 / 理 / 术  │
└──────┬──────┘
       │ 提供识别/判断/执行能力
       ▼
┌─────────────┐     ┌──────────────┐
│  即时遭遇系统  │────→│ 失败累积系统    │
│ 快层 + 慢层   │     │ debuff → 撤退  │
└──────┬──────┘     └──────────────┘
       │ 消耗
       ▼
┌─────────────┐     ┌──────────────┐
│  身体资源系统  │────→│ 预制道具系统    │
│  血 + 尿      │     │ 安全区制作复合品 │
└──────┬──────┘     └──────────────┘
       │ 恢复/补给
       ▼
┌─────────────┐     ┌──────────────┐
│   安全区活动   │────→│ 阅读引导系统    │
│ 经济/社交/学习 │     │ 遭遇笔记/交叉发现│
└─────────────┘     └──────────────┘
```

## 十二、与现有系统的关系

本设计基于现有 GameDraft 架构，核心依赖：
- **FlagStore + 条件系统**：遭遇触发条件、规矩状态查询
- **规矩系统（RulesManager）**：象/理/术三层、碎片收集、与FlagStore同步
- **遭遇系统（EncounterManager）**：扩展支持即时遭遇的快层/慢层模式
- **道具系统（InventoryManager）**：道具消耗、快捷栏绑定
- **区域系统（ZoneSystem）**：危险区域标记、遭遇触发概率
- **ActionExecutor**：统一执行遭遇结果动作

新增系统：身体资源系统（BodyResourceSystem）、即时遭遇管理器（TemplateEncounterManager）、失败累积管理、遭遇笔记系统、预制道具系统。
