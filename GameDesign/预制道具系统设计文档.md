# 预制道具系统设计文档

> 术层的复合操作在安全区提前制作，不在遭遇现场合成。
> 上级文档：《遭遇系统总设计文档》

---

## 一、设计定位

预制道具系统解决一个核心问题：**术层规矩（如「血画引火符」）需要组合多种材料，但遭遇现场不允许慢慢合成。**

解法：在安全区提前制作好复合道具，带进危险区直接使用。制作本身消耗身体资源（血）和道具，是一种前置投资。

---

## 二、制作流程

### 2.1 前置条件

- 玩家必须已解锁对应规矩的 **术层**
- 拥有所需的原材料道具
- 身体资源足够（如需要放血，则血量需高于安全线）
- 必须在安全区域内进行

### 2.2 制作步骤

1. 打开制作界面（在安全区可随时打开）
2. 选择要制作的配方（只显示已解锁术层的配方）
3. 确认材料消耗和身体资源消耗
4. 执行制作（可能有简单的交互，如画符的方向选择——待定）
5. 获得成品道具，放入背包

### 2.3 制作消耗

每个配方的消耗在数据中定义：

```
配方定义 {
  id: 配方唯一标识
  name: 成品名称
  requiredRule: 需要的规矩ID
  requiredLayer: "shu"（必须是术层）
  materials: [
    { itemId: 原材料道具ID, count: 数量 }
  ]
  bloodCost: 血量消耗（0表示不需要血）
  urineCost: 尿量消耗（0表示不需要尿）
  bloodQualityRequired: 需要的血质属性（可选）
  resultItem: {
    itemId: 成品道具ID
    count: 产出数量
  }
  craftTime: 制作时间（游戏内时间消耗，可选）
}
```

## 三、配方示例

| 成品 | 所需规矩术层 | 材料消耗 | 身体消耗 | 效果 |
|------|------------|---------|---------|------|
| 引火符木 | 白毛僵尸怕火·术 | 桃木片x1 | 少量血 | 对僵尸/鬼类确定有效的一次性道具 |
| 血符纸 | （待定驱鬼术·术） | 符纸x1 | 中等血 | 对厉鬼有效的强化符纸 |
| 童子尿瓶 | （待定净化术·术） | 空瓶x1 | 中等尿 | 可存储的尿，延长尿的保质期 |
| 药糯米 | （待定驱僵术·术） | 糯米x1 + 草药x1 | 无 | 对僵尸效果大幅增强的糯米 |

注：具体配方随规矩内容扩展而增加。以上仅为示例。

## 四、成品道具的使用

### 4.1 在即时遭遇中使用

预制成品可以：
- 绑定到快捷栏，在快层中直接使用（和普通道具一样按快捷键）
- 在慢层面板中作为选项出现

### 4.2 效果等级

预制成品的效果通常优于原材料直接使用：

| 使用方式 | 效果 |
|---------|------|
| 直接扔桃木片（无知识） | 成功率 20-30% |
| 有理层知识 + 扔桃木片 | 成功率 50-60% |
| 使用预制「引火符木」 | 成功率 95-100% |

这就是预制的价值：**花费前置投资（血+材料+制作时间），换取遭遇中的确定性。**

### 4.3 在支线解谜链中使用

部分支线的关键节点可能需要特定预制道具才能推进。这在支线数据中通过道具条件（flag/道具检查）实现。

## 五、现场合成（极少数高级规矩）

### 5.1 定位

绝大部分规矩的术层执行要求预制。但极少数高级规矩允许在遭遇现场临时合成，作为一种 **稀缺的高级能力**。

### 5.2 限制条件

- 只有特定的高级规矩标记了「允许现场合成」
- 现场合成的消耗比安全区预制更大（1.5-2倍）
- 现场合成需要在慢层面板中操作（不能快层执行）
- 合成过程可能有额外的操作要求（待定）

### 5.3 设计意图

现场合成是「知识深厚到可以临场应变」的能力体现。玩家遇到一个棘手的遭遇，手上没有预制品，但他学了一条高级规矩可以现场做——这个时刻的满足感来自知识积累的回报。

## 六、与其他系统的交互

| 系统 | 交互方式 |
|------|---------|
| 规矩系统 | 术层解锁 → 配方解锁；配方引用规矩ID和层级 |
| 身体资源系统 | 制作消耗血/尿；检查血质是否满足配方要求 |
| 道具系统 | 消耗原材料、产出成品；成品作为普通道具存入背包 |
| 即时遭遇系统 | 成品道具在快层/慢层中作为高效选项出现 |
| 经济系统 | 原材料需要花铜钱购买 |

---

## 七、数据驱动

所有配方在 JSON 数据文件中定义，不硬编码。新增配方只需：
1. 定义新的规矩（含术层）
2. 在配方数据中添加一条配方
3. 在道具数据中定义成品道具的属性

零代码改动即可扩展。
